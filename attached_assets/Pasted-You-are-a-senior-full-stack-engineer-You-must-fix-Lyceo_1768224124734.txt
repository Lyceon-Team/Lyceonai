You are a senior full-stack engineer. You must fix Lyceon’s intermittent auth/login loop deterministically.

## Context / Symptom
Sometimes Google OAuth succeeds, but user is silently returned to /login or sees an endless auth loop.
Network shows GET /api/auth/user → 401 with {"error":"Invalid or expired token"} or missing cookies.

## Root Cause (must address)
Cookies `sb-access-token` and `sb-refresh-token` are set without an explicit cookie `path`.
In Express, default cookie path becomes the current request path (e.g. /api/auth/google),
so cookies may not be sent to /api/auth/user depending on which auth flow executed.
This creates intermittent login failures.

## Hard requirements
1) Every place we set auth cookies MUST include `path: '/'`.
2) Every place we clear auth cookies MUST include `path: '/'` (or they won’t clear reliably).
3) Access-token cookie maxAge must match token lifetime (expires_in), not 7 days.
4) Refresh-token cookie can remain longer (e.g. 30 days), but must also use path '/'.
5) Provide proofs: grep proof + curl proof + manual browser proof.

function setAuthCookies(res: any, session: any, isProd: boolean) {
  const accessMaxAgeMs =
    typeof session?.expires_in === 'number'
      ? session.expires_in * 1000
      : 60 * 60 * 1000; // fallback 1h

  // Keep refresh longer; adjust if you prefer
  const refreshMaxAgeMs = 30 * 24 * 60 * 60 * 1000; // 30 days

  const base = {
    httpOnly: true,
    secure: isProd,
    sameSite: 'lax' as const,
    path: '/', // CRITICAL
  };

  res.cookie('sb-access-token', session.access_token, {
    ...base,
    maxAge: accessMaxAgeMs,
  });

  res.cookie('sb-refresh-token', session.refresh_token, {
    ...base,
    maxAge: refreshMaxAgeMs,
  });
}

function clearAuthCookies(res: any, isProd: boolean) {
  const base = {
    httpOnly: true,
    secure: isProd,
    sameSite: 'lax' as const,
    path: '/', // CRITICAL
  };
  res.clearCookie('sb-access-token', base);
  res.clearCookie('sb-refresh-token', base);
}

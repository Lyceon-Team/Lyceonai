# ✅ Change 2 — Fix Incorrect Cookie Clearing in apps/api bridge (stale cookie risk)

## Why this matters

There is a signout handler that clears only `sb-access-token` **without path/domain**, which can fail to clear the real cookie (set with `path: '/'` and prod `domain: '.lyceon.ai'`), leaving stale cookies behind.

## File

`apps/api/src/auth/express-bridge.ts`

## Current lines (must verify in repo)

Lines **67–70** currently:

```ts
router.post('/signout', (_req: Request, res: Response) => {
  res.clearCookie('sb-access-token');
  res.json({ ok: true, url: '/login' });
});
```

## Required patch

Update this handler to clear BOTH auth cookies with the same options used elsewhere in prod.

Replace lines 67–70 with:

```ts
router.post('/signout', (req: Request, res: Response) => {
  const isProd = process.env.NODE_ENV === 'production';

  const base = {
    httpOnly: true,
    secure: isProd,
    sameSite: 'lax' as const,
    path: '/',
    ...(isProd && { domain: '.lyceon.ai' }),
  };

  res.clearCookie('sb-access-token', base);
  res.clearCookie('sb-refresh-token', base);

  // Also attempt host-only clear to flush any past mis-scoped cookies
  res.clearCookie('sb-access-token', { ...base, domain: undefined as any });
  res.clearCookie('sb-refresh-token', { ...base, domain: undefined as any });

  res.json({ ok: true, url: '/login' });
});
```

### IMPORTANT

* Do NOT change any other routes in this file.
* Do NOT remove bearer/cookie token support in `/session`.
* This change is purely to prevent stale cookie ghosts.

Run:

```bash
nl -ba apps/api/src/auth/express-bridge.ts | sed -n '55,90p'
```

and paste output showing the updated signout handler.

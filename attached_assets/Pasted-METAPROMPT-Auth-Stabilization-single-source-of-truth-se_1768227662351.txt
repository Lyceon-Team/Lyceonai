Step 1: Freeze the Supabase client from persisting/refreshing sessions
Edit file: client/src/lib/supabase.ts

Current (lines ~18–22):

  autoRefreshToken: true,
  persistSession: true,
  detectSessionInUrl: true,
  flowType: 'pkce',
},


Replace with:

auth: {
  // Server cookies are the source of truth.
  // Do not persist or auto-refresh on the client; it can race with /api/auth/refresh.
  autoRefreshToken: false,
  persistSession: false,

  // We do NOT use client-side OAuth callbacks (we use /auth/google/callback on the server).
  detectSessionInUrl: false,

  flowType: 'pkce',
},


Why: This stops the Supabase JS client from storing and refreshing tokens independently, which can cause refresh-token reuse and infinite loops.

Step 3: Make SupabaseAuthContext rely on backend cookies first, not Supabase session
Edit file: client/src/contexts/SupabaseAuthContext.tsx

In initializeAuth() you currently do:

supabase.auth.getSession() then maybe /exchange-session, then separately fetchUserFromBackend().

Change the initialization order to:

Always attempt backend cookie hydration first:

const backendUser = await fetchUserFromBackend();

If backendUser exists: setUser + stop.

Only if backendUser is null:

attempt supabase.auth.getSession() (for fresh email/password sign-in in same tab only)

if session exists, call /api/auth/exchange-session

then call fetchUserFromBackend() again

Replace your initializeAuth() body with this (drop-in):
const initializeAuth = async () => {
  try {
    // 1) Backend cookie session is the source of truth
    const backendUser = await fetchUserFromBackend();
    if (backendUser) {
      setUser(backendUser);
      return;
    }

    // 2) Only fallback to in-memory Supabase session (email/password flow)
    const { data: { session } } = await supabase.auth.getSession();
    if (session?.access_token) {
      await exchangeSession(session.access_token);
      const refreshedUser = await fetchUserFromBackend();
      if (refreshedUser) setUser(refreshedUser);
    }
  } catch (err) {
    console.error('[AUTH] initializeAuth failed:', err);
  } finally {
    setLoading(false);
  }
};


Also remove or neuter any logic that triggers refresh loops repeatedly. Specifically:

ensure you are not running initializeAuth() multiple times

ensure you don’t call refresh in both an effect and in fetchUserFromBackend()

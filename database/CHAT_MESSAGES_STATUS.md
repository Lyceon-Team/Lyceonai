# Chat Messages Table Status

## Current Status: UNUSED ⚠️

The `chatMessages` table in the PostgreSQL schema (`shared/schema.ts`) is **currently not used** by any API endpoints.

### Findings

1. **No API Routes**: No `/api/chat` routes exist in `server/index.ts`
2. **No Queries**: No code queries the `chatMessages` table
3. **Schema Exists**: The table definition exists in `shared/schema.ts` with userId column
4. **Database Table**: The table exists in PostgreSQL but is empty and unused

### Schema Definition

```typescript
export const chatMessages = pgTable("chat_messages", {
  id: varchar("id").primaryKey().default(sql`gen_random_uuid()`),
  userId: varchar("user_id").references(() => users.id).notNull(),  // ✅ Added for security
  message: text("message").notNull(),
  response: text("response").notNull(),
  sources: jsonb("sources"), // Array of question IDs that were used as context
  timestamp: timestamp("timestamp").defaultNow().notNull(),
});
```

### Security Posture

✅ **Schema is Secure**: The userId column is present and marked NOT NULL  
⚠️ **No Implementation**: Since no endpoints exist, there's no active security risk  
📝 **Future Implementation Must**:
  - Filter reads by `WHERE user_id = req.user.id`
  - Set userId on insert: `userId: req.user.id`
  - Require authentication via `requireSupabaseAuth` middleware

### Recommendations

#### Option 1: Keep for Future Use (Current Approach)
- ✅ Leave schema in place with userId
- ✅ Document that future implementations MUST filter by userId
- ✅ Include in RLS test suite when implemented

#### Option 2: Remove Unused Table
- Remove from `shared/schema.ts`
- Run `npm run db:push` to drop table
- Cleaner codebase, no unused tables

#### Option 3: Implement Now
- Create `/api/chat` endpoints
- Use for AI tutor chat history
- Follow same pattern as practice sessions:
  ```typescript
  // Create message
  await db.insert(chatMessages).values({
    userId: req.user.id,  // Always set from auth
    message,
    response,
    sources
  });
  
  // Read messages
  const messages = await db.select()
    .from(chatMessages)
    .where(eq(chatMessages.userId, req.user.id))  // Filter by user
    .orderBy(desc(chatMessages.timestamp));
  ```

## Decision

**Current Decision**: Keep table with userId for future use

**Rationale**:
1. Schema is already secure (has userId column)
2. No active security risk (table unused)
3. Future implementations will have proper security from the start
4. Removing and re-adding later would require migration

## If Implementing Chat in Future

### Required Steps

1. **Create API Routes** (`server/routes/chat.ts`)
   ```typescript
   import { requireSupabaseAuth } from '../middleware/supabase-auth';
   
   // POST /api/chat/send - Send a message
   export const sendMessage = async (req: AuthenticatedRequest, res: Response) => {
     const { message } = req.body;
     
     // Create chat message with authenticated user ID
     const chatMessage = await db.insert(chatMessages).values({
       userId: req.user.id,  // ← Required
       message,
       response: '', // Generated by AI
       sources: []
     }).returning();
     
     res.json(chatMessage);
   };
   
   // GET /api/chat/history - Get user's chat history
   export const getChatHistory = async (req: AuthenticatedRequest, res: Response) => {
     const messages = await db.select()
       .from(chatMessages)
       .where(eq(chatMessages.userId, req.user.id))  // ← Required
       .orderBy(desc(chatMessages.timestamp));
     
     res.json(messages);
   };
   ```

2. **Register Routes** (`server/index.ts`)
   ```typescript
   import { sendMessage, getChatHistory } from './routes/chat';
   
   app.post('/api/chat/send', requireSupabaseAuth, sendMessage);
   app.get('/api/chat/history', requireSupabaseAuth, getChatHistory);
   ```

3. **Add Tests** (`tests/specs/chat-isolation.spec.ts`)
   ```typescript
   test('should deny cross-user access to chat history', async () => {
     // User A sends message
     await user1Context.request.post('/api/chat/send', {
       data: { message: 'Test message' }
     });
     
     // User B tries to access it
     const response = await user2Context.request.get('/api/chat/history');
     const messages = await response.json();
     
     // Should only see own messages, not User A's
     expect(messages).toHaveLength(0);
   });
   ```
